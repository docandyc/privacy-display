**IEEE Access 投稿规划：基于时间分片掩模的显示器隐私保护系统**

> ▎ 交付说明：本文件是规划文档的完整草稿。经批准后，我会把它写入项目内 `IEEE_Access_submission_plan.md`（与现有 `IEEE_Access_literature_gap_analysis.md` 同级），作为 1 个月冲刺投稿的执行手册。计划阶段不修改任何代码或实验，仅产出本规划文档。

## Context（为什么要做这件事）

设 PoC（privacy-display）已完成大量实验：120 样本语料 × 3 个开源 OCR、强相机攻击基线（相位搜索/差分累加/蓝通道 x）、噪声消融（B1）、重构攻击（B2）、视角攻击（G3）、U-Net 重构（G5）、YOLOv8n 检测、性能实测（A4）、HDR 补偿、多屏同步、可复现 manifest。诚实边界也已暴露：攻击者覆盖完整周期后，时域平均/相位搜索/蓝通道 max 可恢复 94–95% 字符、泄露率 100%——这是纯线性时间分片的原理上界。

但目前几乎所有结论都建立在「模拟相机 + 桌面合成截图 + 开源 OCR」之上。要达到 IEEE Access 录用标准（实验完整、有 baseline 对比、可复现、覆盖 2024–2025 文献、真实信道验证），还差几组关键实验和消融。

**目标：** 用 1 个月冲刺补齐「不补必拒」的实验，把项目从「本科毕设 PoC」抬到「IEEE Access 可送审」，并为每个实验明确对应论文的章节/表/图。

**已确认约束（用户答复）：**
1. 有真实相机设备 + 240Hz+/HDR 屏 + 可联网 VLM key；
2. 无法招募正式人因被试 → 人因实验降级为最小自评 pilot + 局限说明；
3. 时间线 1 个月冲刺；
4. 文档聚焦实验设计 + 论文章节映射。

**投稿定位（一句话，沿用 gap analysis 的结论，避免"首次"过度声称）：**
> ▎ A software-only, camera-resistant display-masking system for smart-glasses-era screen-text privacy, combining cryptographically randomized temporal subframe masking with OCR-oriented complementary perturbation, validated on real cameras and modern OCR/VLM recognizers, with an explicit full-cycle-averaging boundary analysis.

## 3. 缺口消融实验设计（本规划核心）

> ▎ 设计原则：每个实验给出 目的 / 自变量 / 固定量 / 指标(因变量) / baseline / 期望结论 / 复用代码 / 工作量 / 是否环境依赖 / 论文位置。指标统一用项目已有的 `text_recovery_metrics()`：字符恢复率、词级准确率、exact-match、敏感 token 恢复率、泄露率（字符≥20%）；可用性 FPI / CIEDE2000 ΔE / SSIM。

### 实验 A — 真实设备防拍实验【P0 不补必拒｜资源已具备】
- **目的：** 把模拟相机结论迁移到真实相机+真实高刷屏，证明防御不是模拟产物。
- **自变量：** 采集策略{单拍 / 短曝光≤1/500s / 自动曝光长曝光 / 视频流抽帧} × 距离{0.5,1,2 m} × 角度{0,15,30,45°} × n{2,4} × 内容{英文/中文/凭据/代码/表格/URL}。
- **固定量：** 刷新率（按所用高刷屏实际值，记录真实 readout/曝光）、亮度、环境光。
- **指标：** 四项恢复率 + 泄露率；同帧记录人眼可读对照（按 O 键切换原图/拍摄对照）。
- **baseline：** 未保护屏；若有防窥膜则加一组对照。
- **期望结论：** 单拍/短曝光/视频流逐帧 ≈ 0% 恢复；自动曝光长曝光复现"完整周期还原"边界（与模拟一致）。
- **复用：** `experiments/real_capture_analysis.py --init-template` → 采集 → `--engines ...`；`main.py playback`（高刷屏人眼端演示，`--anti-ocr-profile strong/vlm`）。
- **工作量：** 中（采集是瓶颈，分析链路已就绪）。
- **论文位置：** Evaluation §真实设备；主结果 Table + 拍摄对照 Figure。

### 实验 B — 现代识别器 + 在线 VLM 消融【P0 不补必拒｜资源已具备】
- **目的：** 证明"单子帧 0% 防御"不局限开源 OCR，并验证完整周期重构对 VLM 同样泄露（边界一致）。
- **自变量：** 识别器{Tesseract,EasyOCR,PaddleOCR,+TrOCR,+DocTR,+手机系统 OCR(Apple Live Text/Google Lens),+在线 VLM} × 攻击帧{原始 / 单子帧 / 完整周期重构 / 相位 max / 蓝通道 max}。VLM 取 2–3 个模型（Qwen3-VL + 另 1–2 个）。
- **固定量：** 同一批攻击帧、同一 ground truth（VLM 只收图像+转写指令，不发 GT，本地算指标，避免污染）。
- **指标：** 四项恢复率。
- **baseline：** 原始帧每个识别器的基准准确率。
- **期望结论：** 所有识别器单子帧≈0%；完整周期重构对 OCR 与 VLM 均高恢复 → 边界结论跨识别器成立。
- **复用：** 联网重跑 `experiments/vlm_readability_analysis.py`（替换失败诊断文件）+ 扩 `ocr_evaluator.py` 接 TrOCR/DocTR；`publication_summary` 自动收编。
- **工作量：** 中。
- **论文位置：** Evaluation §识别器泛化；Table。

### 实验 C — 组件贡献系统化消融【P0｜复用已有代码】
- **目的：** 把分散的 B1/B2/pedestal 发现整合成一张逐组件消融表，量化"哪个组件在哪个攻击下不可或缺"。
- **自变量（逐步加组件）：** ① 仅掩模 → ② +互补对抗噪声 → ③ +反色帧 → ④ +黑帧&AE 攻击 → ⑤ +噪声 pedestal。每组在 {正常预算 ε=8/255｜弱掩模泄露压力 leak=0.75,ε=96/255} 两场景各跑。
- **固定量：** n=4、语料、刷新率。
- **指标：** 单帧 OCR、单帧 inpaint 重构 OCR、长曝光 OCR。
- **期望结论（已有线索）：** 掩模是主防御（单独压到地板）；pedestal 对单帧 inpaint 不可或缺（去 pedestal → inpaint 还原 100%）；噪声在弱掩模泄露场景提供可测边际（OCR 降幅≈46pct）；黑帧+AE 进一步压长曝光。
- **复用：** `experiments/ablation_noise.py` + `reconstruction_attack` + `camera_simulator.auto_exposure_attack`。
- **工作量：** 低–中（主要是组织成统一表+统计）。
- **论文位置：** Evaluation §消融（核心 Table，论文最有学术价值的部分之一）。

### 实验 D — 感知色彩空间消融【P1｜HDR 屏已具备】
- **目的：** 回答审稿人"为什么用 RGB 随机点阵而不是感知空间调制"，并检查是否引入单通道侧信道。
- **自变量：** 调制空间{RGB 随机点阵(当前) / OKLAB / ICtCp / 蓝通道感知}。
- **指标：** 可用性(ΔE2000/SSIM/FPI) + 防御(单帧 OCR) + 侧信道(蓝通道 max 恢复率)。
- **期望结论：** 感知空间可降可见性，但某些通道可能成为相机稳定侧信道（呼应 DeepLight/Revelio）→给出"既要低可见性又不暴露侧信道"的取舍。
- **复用：** `src/core/hdr_compensation.py` + `metrics.py`。
- **工作量：** 中。
- **论文位置：** Evaluation §感知优化消融。

### 实验 E — 参数扫描消融（安全-可用性 Pareto）【P1｜复用 benchmark】
- **目的：** 给出推荐工作点与 trade-off 曲线。
- **自变量：** n∈{2,4,6,8} × refresh∈{144,240,360} × ε × mask cell size。
- **指标：** 可用性(FPI/ΔE) vs 安全(单帧 OCR & 完整周期泄露率)。
- **期望结论：** 随 n↑ 可用性约束变紧（需更高刷新率），安全趋稳；画出 Pareto 前沿并标推荐点。
- **复用：** `src/evaluation/benchmark.py` 参数扫描。
- **工作量：** 低。
- **论文位置：** Evaluation §参数敏感性；trade-off Figure。

### 实验 F — 强攻击者补强（边界分析）【P1/P2】
- **目的：** 完善边界分析，让"线性时间分片不可能性"对更强攻击者也清晰。
- **自变量：** +rolling-shutter 行对齐重构、+时域超分；对比现有相位/差分/蓝通道 max。
- **指标：** 恢复率 + SSIM/PSNR。
- **复用：** `src/attack/camera_simulator.py`。
- **工作量：** 中。
- **论文位置：** Boundary Analysis §（负面结果 = 贡献）。

### 实验 G — 人因（资源受限→降级）【P2】
- 用户无法招募正式被试 → 做最小自评 pilot（N=3–5，主观闪烁/可读性 5 分量表 + 30 分钟连续观看自评），明确标注样本量不足、不作强结论；正式 10 人级实验写入 Future Work + Limitation。
- **论文位置：** Limitation / Future Work（+ 可选小 pilot 附录）。

## 7. 风险与诚实边界（必须写进论文，避免被一击否定）

1. **完整周期重构泄露：** 正面写成 boundary contribution（线性时间分片不可能性），把安全声明限定在被动抽帧/流式逐帧/短曝光/卷帘快门威胁模型；不宣称防御主动连续录制后的完整周期重构。
2. **性能：** Python PoC 不能宣称"实时系统"；明确生产级实时需 GPU compute shader/合成器/驱动层。
3. **人因样本不足：** 自评 pilot 仅作可用性旁证，正式实验写入 Future Work。
4. **VLM 可复现：** 记录模型版本、抽样范围、调用日期；密钥仅环境变量，不入结果/manifest。
5. **真实设备代表性：** 记录每台相机的 readout/曝光/AE 策略，避免以单一设备外推。
