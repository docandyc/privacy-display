毕设实现规划：基于视觉暂留与时间分片掩模的显示器隐私保护系统

Context

本方案基于技术交底书，实现目标为概念验证原型（PoC），在 60/144Hz 普通显示器上以 Python + GPU 技术栈验证核心算法的有效性。无需内核驱动级注入；重点是：

1. 证明单帧 OCR 准确率可从 ~98% 降至 ~12%
2. 证明时域叠加后可还原原始画面（模拟人眼积分效果）
3. 量化 FPI、Delta E、信息熵等评估指标

硬件约束说明：60Hz 显示器上 n=2 会产生 30Hz 闪烁（肉眼可见），n=4 需 240Hz。PoC 策略是：将应用层子帧切换演示与静态单帧分析分离——用静态测试集验证 OCR 防御效果，用视频合成模拟视觉积分效果（帧混合模拟人眼），无需高刷屏即可验证所有核心指标。

分阶段实现计划（共 6 周）

Phase 1：核心掩模算法（第 1-2 周）

目标：实现随机点阵掩模生成与子帧分解核心逻辑。

文件：src/core/mask_generator.py
- 使用 pycryptodome 的 ChaCha20 作为 CSPRNG
- 生成随机索引矩阵 R(x,y) ∈ {0…n-1}
- 卡方检验（scipy.stats.chisquare）确保均匀性
- 输出 n 个二值掩模矩阵 M_k（numpy bool array）

文件：src/core/subframe_composer.py
- 子帧合成：I_k = I * M_k
- 完备性验证：sum(I_k for all k) == I
- SDR 亮度补偿：I_k_comp = clip(I_k * gamma, 0, 255)
- Fisher-Yates 置换子帧输出顺序

验证：单元测试验证 ΣI_k = I，掩模均匀性 p > 0.01

---
Phase 2：对抗噪声生成（第 2-3 周）

目标：生成对抗噪声并验证时域互补消除性质。

文件：src/core/noise_injector.py
- 使用 PyTorch + FGSM 针对 Tesseract OCR 特征生成基础噪声 N_base（ε=8/255）
- 互补分配：N_1=+N_base/2, N_2=-N_base/2, N_3=+N_base/2, N_4=-N_base/2
- 验证 ΣN_k = 0
- 离线预计算噪声模板库（Tesseract、EasyOCR），存入 data/noise_templates/

文件：src/attack/ocr_evaluator.py
- 对原始图像和单子帧图像分别运行 Tesseract / EasyOCR
- 计算字符准确率（CER）和词准确率（WER）
- 输出对比结果

验证：单子帧 OCR 准确率应从 ~98% 降至 <20%

---
Phase 3：GPU 渲染 Demo（第 3-4 周）

目标：基于 moderngl 实现实时子帧渲染与视觉积分演示。

文件：src/gpu/renderer.py（使用 moderngl）
- 掩模纹理上传至 GPU
- Pixel Shader 实现：I_k_raw = I * M_k + N_k
- 帧缓冲区管理（双缓冲）
- 支持 60Hz/144Hz 下 n=2 实时切换演示

文件：src/demo/visual_integration.py
- 将 n 个子帧按等权叠加还原：I_restored = mean(I_1..I_n)
- 对比原始帧与还原帧（计算 Delta E）
- 生成 GIF/视频展示"人眼看到的"与"相机看到的"对比

文件：src/demo/privacy_window.py（pygame）
- 捕获当前屏幕（mss 库）
- 应用掩模后以子帧序列循环显示
- 144Hz 环境下 n=2 实时运行

---
Phase 4：评估体系（第 4-5 周）

目标：实现全部量化指标，输出论文用数据。

文件：src/evaluation/metrics.py
- FPI（闪烁感知指数）：基于 IEEE 1789-2015，FPI = modulation_depth / (f_r/n * threshold_curve)
- Delta E：使用 colormath 库计算 CIEDE2000，比较原始帧与积分帧
- 信息熵：scipy.stats.entropy 计算单子帧 vs. 原始帧熵值比
- 亮度均匀性：9 区域亮度差异 ΔL/L

文件：src/evaluation/benchmark.py
- 批量测试不同 n（2/4/6）、不同 ε（4/8/12 /255）下的指标
- 输出表格（对应交底书第 6.4 节功耗与性能分析表格）
- 生成论文图表（matplotlib）

---
Phase 5：抗多帧叠加验证（第 5 周）

目标：验证攻击者即使叠加多帧也无法恢复信息。

文件：src/attack/camera_simulator.py
- 模拟攻击者采集 k 帧并做时域平均（Temporal Averaging）
- 测试叠加 2/4/8/16 帧后的 OCR 准确率
- 验证对抗噪声的互补抵消机制使叠加无效

验证：多帧叠加后 OCR 准确率应 <30%（不受噪声互补消除影响是因为掩模随机性不同）

---
Phase 6：整合与文档（第 6 周）

- 完善交互式 Demo（带 GUI 的演示界面）
- 整理实验结果，生成论文所需图表
- 撰写技术文档

验证方案

1. 算法正确性：pytest tests/ 验证 ΣI_k=I、ΣN_k=0、掩模均匀性
2. 防御效果：运行 benchmark.py 输出 OCR 准确率对比表
3. 视觉质量：visual_integration.py 生成对比图，目标 Delta E < 1.0
4. 性能开销：计时 GPU 着色器执行，验证 <1ms 帧生成时间
5. Demo 展示：在 144Hz 显示器上运行 privacy_window.py (n=2)，屏幕录制展示效果
