## 摘要（Abstract）

**问题**：智能手机和智能眼镜（如 Ray-Ban Meta）可以静默拍摄屏幕，从照片中通过 OCR、目标检测、VLM 等手段窃取密码、金融数据、个人敏感信息。这是一种被低估的物理层隐私威胁。

**现有不足**：物理隐私膜只限制视角，无法阻止正对相机的窃取；屏幕调暗和数字水印是事后取证手段而非事前阻断；软件打码虽然有效但牺牲了正常使用体验。

**方法**：我们提出一种基于视觉暂留（persistence of vision）的时域像素掩模显示方法。将每帧画面在时域上分解为 n 个随机互补子帧（CSPRNG 驱动），叠加互补对抗噪声（积分后精确归零）。人眼在其 ~50ms 时间积分窗口内自然叠加所有子帧，感知完整画面；相机在任意快门时刻只能捕获单个信息熵极低的子帧碎片，OCR 和检测器均无法识别。

**核心证据**：在 10,575 张真机拍摄中，按攻击者可选择的最优 OCR 引擎统计，短曝光字符恢复率从未保护画面的 92.5% 降至 deployed 档的 14.1% 和抗拍强化档的 4.2%，exact-match 从 66.0% 降至 0.4% / 0%。在受控语料上，三个 OCR 引擎的识别率从 94% 降至 0%。COCO val2017 上四个检测器的 mAP50 从 57–68% 降至 0.4–4.3%。MOT17 跟踪指标崩塌。FPI=0.030、ΔE=0.37。

**边界**：视频时域平均攻击是残余前沿（完整周期积分可恢复 95%）。抗拍强化档在真机视频上仍有 42.3% char / 4.6% exact，因此只能声称缓解，不能声称完全阻断。[TODO: 用户研究补做后补充主观舒适度数字]

**关键词**：display privacy, visual eavesdropping, camera capture, persistence of vision, temporal masking, adversarial perturbation, OCR, anti-photography

## I. 引言（Introduction）

### 1.1 威胁背景（钩子）

屏幕无处不在——办公、金融、医疗、会议。与此同时，拍照设备同样无处不在：手机、智能眼镜、微型相机。攻击者只需对屏幕拍一张照片，就能通过 OCR 提取文字、通过目标检测识别界面元素、甚至通过 VLM（视觉语言模型）直接"读懂"屏幕内容。这种"视觉窃听"（visual eavesdropping）攻击简单、隐蔽、难以察觉——没有网络入侵痕迹、不需要物理接触设备。

### 1.2 现有方案及其不足

| 方案类别 | 代表 | 防护能力 | 致命缺陷 |
|---------|------|---------|---------|
| 物理隐私膜 | 3M 隐私滤光片 | 限制可视角度 | 正对相机毫无防御 |
| 亮度降低 | 调暗屏幕 | 增加拍摄难度 | 实测 OCR 仍达 92.9% |
| 高斯模糊 | 软件层 | 模糊后 OCR 降至 52.6% | 正常用户也看不清 |
| 像素化打码 | 马赛克 | OCR 降至 0% | 正常用户也读不到 |
| 数字水印 | 隐写/追踪 | 事后追责 | 不阻止信息泄露本身 |

**核心矛盾**：现有方案要么无法阻止相机窃取，要么在阻止相机的同时也破坏了人眼的正常阅读。

### 1.3 核心思想

利用人眼视觉系统和相机成像系统之间的一个根本性差异：

- **人眼**：在约 50ms 的时间窗口内对光信号做时间积分，实现视觉暂留（persistence of vision）。
- **相机**：每次快门只捕获一个瞬时画面。

我们将这一差异转化为防御：把每帧完整画面在时域上拆成多个随机互补碎片，高速交替显示。人眼积分后看到完整画面；相机拍到的任何一帧都是无法识别的碎片。

## III. 威胁模型（Threat Model）

### 3.1 保护资产

屏幕上显示的敏感文字和视觉信息（密码、金融数据、PII、代码、机密文档等）。

### 3.2 攻击者能力（从弱到强）

| 等级 | 攻击方式 | 典型场景 |
|-----|---------|---------|
| L1 | 被动快照：单张高清照片 | 手机偷拍、智能眼镜连拍 |
| L2 | 流式逐帧识别 | 实时 OCR app |
| L3 | 卷帘/全局快门差异利用 | 工业相机 |
| L4 | 长曝光/短曝光 | 手动曝光模式 |
| L5 | 离轴拍摄 | 侧面偷拍 |
| L6 | **连续录像 + 时域平均** | 录制视频后离线处理 |
| L7 | **相位搜索 / 通道分离重构** | 知道周期 n 的自适应攻击者 |

### 3.3 攻击者知识

- **已知**：使用了时域掩模方案、子帧数 n
- **未知**：每帧的具体掩模种子（CSPRNG 密钥）、当前帧的周期相位起点

### 3.4 防护目标与判据

使攻击者获取的图像满足：
- OCR：字符恢复率（char recovery）< 5%，exact match = 0%，敏感 token 恢复率 ≈ 0%
- 检测：mAP50 降至接近 0
- 跟踪：MOTA 降至接近 0
- VLM：逐行 exact-match = 0%

同时保证人眼端：可正常阅读，FPI < 0.1，ΔE < 1.0。

## IV. 方法（Method / System Design）

### 4.1 时域子帧分解

给定原始帧 I，生成 n 个互补子帧 I₁, I₂, ..., Iₙ，满足：

- **完备性**：ΣIₖ = I（人眼积分还原完整画面）
- **互斥性**：每个像素在 n 个时隙中恰好被点亮一次

掩模生成采用 ChaCha20 CSPRNG + Fisher-Yates 置换算法，对每个像素随机分配到某个子帧。掩模的空间随机性通过卡方检验验证。

### 4.2 互补对抗噪声

在子帧上叠加对抗噪声 N₁, N₂, ..., Nₙ，满足：

- **互补性**：ΣNₖ = 0（人眼积分后噪声完全抵消）
- **单帧最大化混淆**：每个子帧上的噪声最大化 OCR 混淆度

噪声分量采用 FGSM 方向并做 Fisher-Yates 时域互补分解。由于屏幕无法显示负光（黑像素处的负噪声会被裁剪），引入基底电平（pedestal）ε，在积分时扣除，代价是黑位抬升 ε/255 的微小对比度损失。

**主次关系**：随机点阵掩模是主防御（单独已将单帧 OCR 压至 0%），对抗噪声是二级强化，用于弱掩模泄露、面板串扰、自适应攻击等极端场景。

### 4.3 亮度补偿与视觉积分

由于每个子帧只包含 1/n 的像素，直接显示会导致亮度降低。采用背光增益模型：背光亮度提升 n 倍（B=n），子帧像素值不变，积分后亮度还原。

色彩保真度由 CIEDE2000 ΔE 度量。HDR 场景的 PQ/HLG 色调映射作为数值扩展实现（注：当前 PoC 非真实 HDR 输出链路）。

### 4.4 时序与带宽

显示刷新率要求：f_refresh ≥ 60 × n（如 n=4 需 ≥240Hz）。

随机掩模的空间相位随机化使人眼感受野汇聚后的调制深度按 1/√N 衰减——这是随机掩模优于全屏高频闪烁方案的数学依据，也是 FPI 模型的核心。

### 4.5 抗拍强化档

基础的线性时域掩模在完整周期时域平均攻击下会泄露（见 §5.4 安全边界）。为应对此前沿攻击及 VLM 版面重建，引入非线性增强档：

- **strong 档**：轻量条纹叠加（stripe α=0.10）+ 字形扰动（glyph α=0.12），6 个子帧周期
- **抗拍强化档（`capture_hardened`）**：更大 cell 尺寸、更密条纹、文字邻域伪笔画网格，用于降低拍照后 OCR/VLM 的机器可读性。历史实验原始数据中的 `vlm` 标签指同一档位。

**trade-off**：非线性档偏离了"严格数学完备性"（ΣIₖ ≠ I），画面会有可感知的条纹/颗粒，但在实际防偷拍场景中这是合理的安全-可用性取舍。

### 4.6 实现范围

当前 PoC 实现应用/算法层（Python + GPU 渲染），不含 OS 驱动级注入（DXGI / KMS-DRM / CoreDisplay 帧缓冲拦截）。后者需内核模块与代码签名，属未来工作。

### 5.3 受控 OCR 机理分析

> **This experiment tests whether**: 单子帧的 OCR 归零是系统性的（跨引擎、跨内容类型），还是特定条件的偶然。

**→ 表 T2：三引擎 OCR 防护效果（120 语料）**

| OCR 引擎 | 样本数 | 原始帧准确率 | 单子帧准确率 | 配对降幅 |
|---------|------|------------|------------|---------|
| Tesseract | 120 | 94.0% (92.3–95.5%) | 0.0% (0.0–0.1%) | 93.9% |
| EasyOCR | 120 | 94.1% (92.7–95.5%) | 0.0% (0.0–0.0%) | 94.1% |
| PaddleOCR | 120 | 94.9% (93.7–96.2%) | 0.0% (0.0–0.0%) | 94.9% |

**分层统计**（以 Tesseract 为主引擎）：

- **按内容类别**：代码/金融/凭证/数字/段落/短文/表格/URL/token，均降至 0.0–0.2%
- **按语言**：英文/中文/代码/符号/混合，均降至 0.0–0.1%
- **按版式**：单栏/多栏，均降至 0.0–0.1%
- **按字号**：17–38pt 全范围，均降至 0.0–0.6%

**结论**：单子帧 OCR 归零是系统性的，不依赖特定引擎、内容或排版。这从机理上解释了 §5.2 中真机快照防护的有效性。

## VI. 讨论（Discussion）

### 6.1 线性积分的数学张力

本方案利用的是人眼积分 vs 相机单帧采样的差异。然而，人眼积分本质上是线性求和（ΣIₖ = I），而相机如果有能力做同样的多帧累加，也能还原原始信息。这是所有基于视觉暂留的时域方案共有的原理性边界。

§5.4 的强攻击评测诚实量化了这一边界：完整周期积分可恢复 95.2%。这不是实现缺陷，而是方案原理的内在张力。

### 6.2 抗拍强化档的作用与代价

抗拍强化档通过条纹叠加和字形扰动打破了严格完备性（ΣIₖ ≠ I），使得完整周期积分难以精确还原——代价是人眼端会感知到轻微的条纹和颗粒。

这是一个务实的安全-可用性取舍：在大多数防偷拍场景中，轻微的视觉降级是可接受的，而信息泄露是不可接受的。§5.2 的真机结果证明了这一取舍的实际有效性。

### 6.3 实际部署路径

当前 PoC 实现为应用层（Python + GPU 渲染），实际部署需要：

- **OS 驱动级帧缓冲注入**：DXGI（Windows）/ KMS-DRM（Linux）/ CoreDisplay（macOS）
- **背光硬件协同**：背光亮度提升 n 倍
- **高刷新率面板**：≥240Hz（n=4）或 ≥360Hz（n=6）

这些均为工程集成层面，不改变核心算法。

### 6.4 伦理声明

本研究是防御性的：旨在保护屏幕内容免受未授权拍摄，而非协助攻击或监控。攻击分析（§5.4）的目的是诚实评估安全边界，而非提供攻击指南。

### 6.5 局限性

1. **视觉无感性验证有限**：目前仅由客观指标（FPI、ΔE）支撑，尚未完成受控用户研究（见 §VIII 占位）。
2. **PoC 非产品级**：应用层实现，未集成 OS 驱动和硬件背光。
3. **HDR 为数值模型**：PQ/HLG 补偿是数学验证，非真实 HDR 输出链路。
4. **真机样本量**：部分拍摄条件的样本量有限（n=5），统计结论需审慎解读。

---

## VII. 结论（Conclusion）

本文提出了一种基于视觉暂留的时域像素掩模显示隐私保护方法，利用人眼时间积分与相机单帧采样的本质差异，实现"人眼可见、相机不可读"的非对称显示特性。

核心结果：

- 在 10,575 张真机拍摄上，短曝光 OCR 字符恢复率从 92.5% 降至 deployed 档的 14.1% 和抗拍强化档的 4.2%。
- 在受控评测上，三个 OCR 引擎的识别率从 94% 一致降至 0%；四个目标检测器的 COCO mAP50 从 57–68% 降至 0.4–4.3%；MOT17 跟踪指标崩塌。
- 视觉无感性：FPI=0.030，ΔE=0.37。
- 完整周期时域平均是数学边界；抗拍强化档可降低恢复率，但未闭合该攻击。

本方案为屏幕隐私保护提供了一条不牺牲正常使用体验的主动防御路径。
